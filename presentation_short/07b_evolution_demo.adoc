=== Ch5 — V1 vs V2
==== One Routing Mechanism, Two Client Options

---

```bash
# Flat — speakerId only, needs a 2nd call
curl .../api/v1/sessions | jq '.[0].speakerId'   # "spk-duke"

# Embedded speaker object — no 2nd call
curl .../api/v2/sessions | jq '.[0].speaker.id'  # {id,name,company}

# Same V2 response, via header
curl -H "X-API-Version: 2" .../api/sessions
```

* A `@PreMatching` `HeaderVersionFilter` rewrites header requests onto `/v2/`.
* V1 & V2 are separate DTOs from the *same* domain object.
