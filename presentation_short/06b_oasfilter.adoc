=== Ch4 — OASFilter
==== Cross-Cutting Spec Rules in One Place

---

*Problem*: some standards apply to _every_ endpoint — repeating annotations on 20+ methods is fragile.

*Solution*: the runtime calls your `OASFilter.filterOperation()` for every operation.

* Add `X-Request-Id` to every response
* Add a global 500 response
* Enforce org-wide standards centrally

```properties
mp.openapi.filter=com.mehmandarov.confapi.openapi.ConferenceOASFilter
```
