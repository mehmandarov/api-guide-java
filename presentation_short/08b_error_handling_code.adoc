=== Bonus – ExceptionMapper
==== No `try/catch` in Resources. Ever.

---

[source,java]
----
@Provider
public class ConstraintViolationExceptionMapper
        implements ExceptionMapper<ConstraintViolationException> {
    @Override
    public Response toResponse(ConstraintViolationException ex) {
        ProblemDetail problem = ProblemDetail.of(400, "Validation Failed")
            .withType("urn:problem-type:validation-error")
            .withExtension("violations", toViolations(ex));
        return Response.status(400).type("application/problem+json")
                .entity(problem).build();
    }
}
----

* One mapper per category: 400 / 404 / 401 / 403 + a `CatchAll` → 500.
* `CatchAll` *never leaks stack traces*. Throw → the mapper handles the rest.
