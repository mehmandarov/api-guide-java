=== Ch5 – The Evolution
==== API Versioning Without the Pain

---

*Versioning is inevitable* – the question is how painfully you do it.

[cols="1,3,3"]
|===
| Strategy | Pros | Cons

| *URI-based* +
`/api/v1/sessions`
| ✅ Explicit, cacheable, easy to test
| ❌ Multiple URLs per resource

| *Header-based* +
`X-API-Version: 2`
| ✅ Clean URIs
| ❌ Harder to test, cache-unfriendly

| *Media type* +
`Accept: …v2+json`
| ✅ Self-describing
| ❌ Complex negotiation
|===

[.lead]
Start with URI versioning. Add header/media-type if clients demand it.
