=== Ch5 — The Evolution
==== API Versioning — Three Strategies

---

*Versioning is inevitable* — the question is how painfully you do it.

[cols="1,3,3"]
|===
| Strategy | Pros | Cons

| *URI-based* +
`/api/v1/sessions` +
`/api/v2/sessions`
| ✅ Explicit, cacheable, easy to test
| ❌ Multiple URLs for same resource

| *Header-based* +
`X-API-Version: 2`
| ✅ Clean URIs
| ❌ Harder to test, cache-unfriendly

| *Media type* +
`Accept:` +
`application/vnd.confapi.v2+json`
| ✅ Clean URIs, self-describing
| ❌ Complex content negotiation
|===

*Opinion*: Start with URI versioning. Add header or media type if clients demand it.
