=== Ch5 — Version Routing
==== `@PreMatching` Filter — Transparent URI Rewriting

---

link:../src/main/java/com/mehmandarov/confapi/versioning/HeaderVersionFilter.java[HeaderVersionFilter.java]

[source,java]
----
@Provider
@PreMatching
public class HeaderVersionFilter implements ContainerRequestFilter {

    @Override
    public void filter(ContainerRequestContext ctx) {
        String path = ctx.getUriInfo().getPath();
        if (path.matches("v\\d+(/.*)?")) return;  // Already versioned

        String version = detectVersion(ctx);       // X-API-Version or Accept
        if (version != null) {
            String newPath = "v" + version + "/" + path;
            ctx.setRequestUri(baseUri, newUri);    // Transparent rewrite
        }
    }
}
----

*Why combine URI + header?* Resource classes just live at `/v1/` and `/v2/`. The filter normalizes header-versioned requests to the same URI paths — one routing mechanism, two client options.
