=== Bonus – Binary Uploads
==== Receiving Files the Portable Way

---

*Two standard shapes*, zero runtime-specific code:

link:../src/main/java/com/mehmandarov/confapi/upload/UploadResource.java[UploadResource.java]

[source,java]
----
@POST @Path("/multipart")
@Consumes(MediaType.MULTIPART_FORM_DATA)
public Response upload(List<EntityPart> parts) throws IOException {
    for (EntityPart part : parts) {
        switch (part.getName()) {
            case "description" -> part.getContent(String.class);
            case "file"        -> part.getContent();   // InputStream – streamed, not buffered
        }
    }
}

@POST @Path("/raw")
@Consumes(MediaType.APPLICATION_OCTET_STREAM)
public Response uploadRaw(InputStream body,
        @HeaderParam("X-File-Name") String fileName) { ... }   // body IS the file
----

* *Multipart* – `List<EntityPart>` (Jakarta REST 3.1+) – text + binary parts in one request
* *Raw* – `application/octet-stream` – the body is the file, metadata travels in headers
* *Quarkus flavour* – `@RestForm` + `FileUpload` in link:../snippets/QuarkusUploadResource.java[snippets/QuarkusUploadResource.java] (reference only – not in the default build)
